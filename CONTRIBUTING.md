# Development Guidelines

When contributing:

- Keep solutions production-ready
- Prefer simplicity over unnecessary complexity
- Document every significant feature
- Follow cloud-native engineering practices
- Keep architecture diagrams updated
- Test examples before submitting
- Maintain consistency across documentation

---

# Pull Request Process

1. Fork the repository.
2. Create a new feature branch.

```
feature/my-improvement
```

3. Commit your changes.

```
git commit -m "Add Kubernetes autoscaling documentation"
```

4. Push your branch.

```
git push origin feature/my-improvement
```

5. Open a Pull Request.

---

# Commit Message Convention

Use clear and descriptive commit messages.

Examples:

```
Add AI Infrastructure architecture

Improve Terraform documentation

Fix Kubernetes deployment example

Update observability diagrams

Refactor README navigation
```

Avoid generic messages like:

```
Update

Changes

Fix

Testing
```

# Reporting Issues

If you discover a bug or have an improvement suggestion:

1. Search existing Issues.
2. Create a new Issue if necessary.
3. Include:
   - Description
   - Steps to reproduce
   - Expected behavior
   - Screenshots (if applicable)
   - Suggested improvements
