# Code of Conduct

## Our Commitment

CloudOps Velocity is committed to fostering an open, professional, respectful, and inclusive engineering community.

We believe that great engineering is built on collaboration, continuous learning, constructive feedback, and mutual respect.

We are dedicated to providing a welcoming environment for everyone, regardless of experience level, background, identity, or technical expertise.

---

# Our Standards

Examples of behavior that contributes to a positive environment include:

- Being respectful and professional
- Welcoming new contributors
- Sharing knowledge openly
- Giving and receiving constructive feedback
- Respecting different technical opinions
- Focusing discussions on solving engineering problems
- Maintaining high standards of documentation and code quality
- Helping improve the platform for everyone

Examples of unacceptable behavior include:

- Harassment or discrimination of any kind
- Personal attacks or abusive language
- Trolling or inflammatory comments
- Publishing confidential information
- Spam or promotional content unrelated to the project
- Disruptive or intentionally harmful contributions
- Any behavior that creates an unsafe or unprofessional environment

---

# Engineering Principles

CloudOps Velocity follows a set of engineering principles that guide every contribution.

- Engineering over complexity
- Automation over repetition
- Reliability over shortcuts
- Security by design
- Cloud-native thinking
- Continuous improvement
- Documentation as a first-class deliverable
- Knowledge sharing across the community

---

# Contribution Expectations

Contributors are encouraged to:

- Write clean and maintainable code
- Follow repository conventions
- Document new features
- Test changes before submitting
- Keep pull requests focused and easy to review
- Respect reviewer feedback
- Help improve existing documentation when possible

---

# Enforcement Responsibilities

Project maintainers are responsible for:

- Reviewing contributions fairly
- Providing constructive feedback
- Maintaining a respectful environment
- Taking appropriate action when this Code of Conduct is violated

Maintainers may edit, reject, or remove contributions that do not align with this Code of Conduct or the project's engineering standards.

---

# Scope

This Code of Conduct applies to all project spaces, including:

- GitHub repositories
- Issues
- Pull Requests
- Discussions
- Documentation
- Project communications
- Community interactions related to CloudOps Velocity

---

# Reporting Issues

If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainers.

**Email**

📧 info@cloudopsvelocity.com

All reports will be handled respectfully and confidentially where possible.

---

# Enforcement

Project maintainers reserve the right to remove comments, issues, pull requests, or contributions that violate this Code of Conduct.

Repeated or serious violations may result in temporary or permanent removal from participation in the project.
